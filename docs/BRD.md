# BRD — 약속잡기 (Business Requirements Document)

**작성일**: 2026-05-13  
**프로젝트**: yaksoklist  
**배포 URL**: https://yaksoklist.vercel.app

## 1. 비즈니스 배경

소규모 모임(친구, 동료, 동아리 등)에서 여럿이 참석 가능한 날짜를 조율하는 과정은 카카오톡·문자 등 메신저에서 수동으로 이루어지고 있다. 참여자가 5명 이상이 되면 개별 응답을 취합하고 최적 날짜를 찾는 것이 번거롭고 시간이 소요된다.

## 2. 비즈니스 목표

| 목표 | 내용 |
|---|---|
| 일정 조율 자동화 | 참여자의 가능 날짜를 한 화면에 집계해 최적 날짜를 자동 추천 |
| 진입 장벽 최소화 | 회원가입·로그인 없이 링크 하나로 즉시 참여 가능 |
| 크로스 디바이스 공유 | 모임장이 생성한 링크를 공유하면 어떤 기기에서도 동일 데이터 접근 |

## 3. 이해관계자

| 역할 | 설명 |
|---|---|
| 모임장 (생성자) | 약속을 만들고 링크를 참여자에게 공유하는 사람 |
| 참여자 | 링크를 통해 접속해 자신의 가능 날짜를 입력하는 사람 |

## 4. 비즈니스 요구사항

### BR-01 인증 없는 접근
로그인·회원가입 없이 링크만으로 약속에 접근할 수 있어야 한다.  
*이유*: 공유 받은 사람이 가입 절차를 거쳐야 하면 참여율이 낮아진다.

### BR-02 크로스 디바이스 데이터 동기화
모임장이 PC에서 만든 약속을 참여자가 모바일로 링크를 열어도 동일하게 보여야 한다.

### BR-03 모임장의 약속 목록 유지
모임장이 여러 약속을 만들었을 때, 기기 내 목록이 유지되어야 한다.

### BR-04 일정 조율 방식 선택
날짜만 조율하는 경우와 시간대(점심/저녁)까지 조율하는 경우를 구분해 지원해야 한다.

### BR-05 최적 날짜 자동 추천
전원 참석 가능한 날짜, 또는 가장 많이 선택된 날짜를 자동으로 계산해 표시해야 한다.

## 5. 비즈니스 제약

- **예산**: 무료 티어만 사용 (Vercel Hobby, Supabase Free)
- **기술 스택**: 별도 서버 운영 없이 Vercel 서버리스 + Supabase 활용
- **유지보수**: 단일 개발자 바이브코딩 방식 운영

## 6. 성공 지표

| 지표 | 목표 |
|---|---|
| 약속 생성 후 1회 이상 참여자 추가율 | 70% 이상 |
| 공유 링크 클릭 후 일정 입력 완료율 | 50% 이상 |
| 페이지 로드 후 3초 내 렌더링 | 99% 이상 |

## 7. 범위 외 (Out of Scope)

- 알림(푸시, 이메일) 기능
- 실시간 동기화 (현재: 새로고침 시 최신 데이터 fetch)
- 구글 캘린더 등 외부 캘린더 연동
- 유료 플랜 / 팀 기능
